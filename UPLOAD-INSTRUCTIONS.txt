BLISKO v1.2.2 - ADAPTIVE SUPPORT UPDATE

This is a complete cumulative GitHub Pages package. It includes the earlier
progress dashboard, answer-visibility safeguards, tutor fixes, progressive
hints, the black-screen repair, and this update.

WHAT CHANGED
- Repeated hint use is remembered per word, phrase, and sentence pattern.
- A difficult typed sentence returns as word ordering; repeated heavy support
  can make it a Polish multiple-choice task.
- Independent successful answers gradually remove the scaffolding again.
- The hint button is kept inside Android's visible viewport when the keyboard
  opens, and asking for a hint no longer forces the keyboard to reopen.
- The completed-session screen now has one clear, full-width Return to Today
  action instead of duplicate small buttons.

UPDATE WITHOUT LOSING PROGRESS
1. Keep the same GitHub repository and GitHub Pages address.
2. Upload every file from this folder directly to the repository root.
3. Allow GitHub to replace matching files and commit to main.
4. Wait for Pages to deploy, then open the normal Pages URL in Chrome while
   online and refresh once.
5. Fully close and reopen the installed app. A second reopen can be needed for
   the new service worker cache.

Do not clear Chrome site data and do not uninstall the PWA. The IndexedDB name,
storage key, manifest id, scope, and Pages path are unchanged, so existing
progress remains available. Export a backup from Settings before updating as
an extra precaution.
