BLISKO v1.2.1 — BLACK-SCREEN REPAIR

CAUSE FOUND
Your repository contains app.js from the progressive-hints update, but engine.js, data.js, storage.js, styles.css, index.html and sw.js are still v1.1.3. The new app asks engine.js for hint functions that the old file does not contain, so JavaScript stops before the page can render. icon-192.png is also missing from the repository.

HOW TO REPAIR WITHOUT LOSING PROGRESS
1. Extract this ZIP on your phone.
2. In the ROOT of your EXISTING GitHub repository, choose Add file > Upload files.
3. Select EVERY extracted item, including .nojekyll and icon-192.png.
4. Commit to main and wait for GitHub Pages to finish deploying.
5. Open your normal GitHub Pages address in Chrome while online.
6. Refresh twice, then fully close and reopen the installed app.
7. If Blisko shows an App Repair panel, tap Repair app cache.

DO NOT create a new repository, uninstall the app, or clear Chrome site data.
The repair button removes only obsolete app-shell caches. It does not touch IndexedDB or localStorage. The database name, state key, manifest ID, scope and website address are unchanged, so your learning progress remains available.
