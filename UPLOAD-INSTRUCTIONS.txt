BLISKO v1.3.0 - TRUST AND MEASUREMENT UPDATE

This is a complete cumulative GitHub Pages package. It includes every earlier
progress, tutor, hint, adaptive-support, and cache-repair update.

WHAT CHANGED
- Polish-aware answer evaluation accepts valid alternatives, flexible word
  order, gender variants, and missing diacritics while explaining real errors.
- Reading, listening, guided production, free production, and pronunciation
  now have separate evidence and confidence histories.
- A short placement and recalibration test creates a conservative CEFR baseline.
- Progress now shows evidence confidence and practical can-do abilities.
- Automatic rotating safety copies are created before migrations and daily.
- App Health can inspect storage, make or restore a safety copy, and safely
  repair downloaded app files without deleting learning data.
- The service worker installs app files as one verified build to avoid mixed
  versions and black screens after updates.

UPDATE WITHOUT LOSING PROGRESS
1. Keep the same GitHub repository and GitHub Pages address.
2. Upload every file from this folder directly to the repository root.
3. Allow GitHub to replace matching files and commit to main.
4. Wait for Pages to deploy, then open the normal Pages URL in Chrome while
   online and refresh once.
5. Fully close and reopen the installed app. A second reopen can be needed for
   the verified service worker build to activate.

Do not clear Chrome site data and do not uninstall the PWA. The IndexedDB name,
storage key, manifest id, scope, and Pages path are unchanged, so existing
progress remains available. Blisko also creates a local safety copy before
schema or version migrations.
